# JEAC Enterprise 5.0：台股／美股分析與報告規範

> 本規範將「個股分析圖表製作提示詞 v1.2」整併為 JEAC Enterprise 5.0 的可執行資料與報告契約。它規範資料、計算、風險揭露與呈現方式；不構成投資建議或報酬保證。

## 1. 市場、代碼與語言

- 台股上市使用「代碼.TW」，上櫃使用「代碼.TWO」；例如 2330.TW、6488.TWO。不可把兩者混用。
- 美股使用交易所通用 ticker，例如 AAPL、NVDA。
- 台股報告使用繁體中文及台灣市場用語；所有現價、收盤價、交易量與技術指標均須附資料日期／時點。
- 系統只產生分析與條件式情境，不自動下單；不得以確定語氣預言價格。

## 2. 資料來源與市場隔離

| 用途 | 台股 | 美股 | 處理原則 |
|---|---|---|---|
| 日線／即時報價 | Fugle（已設定金鑰時）→ Yahoo Finance 備援 | Finnhub／Alpha Vantage／Longbridge（已設定時）→ Yahoo Finance | 不跨市場套用來源。 |
| 官方公告／財報／月營收 | TWSE／TPEx／MOPS | SEC、公司 IR 與交易所公告 | 公告優先於媒體轉載。 |
| 法人／融資券／籌碼 | TWSE／TPEx 等可驗證來源 | 依授權資料源提供情況 | 缺資料時標示，不推估。 |
| 市場複盤 | 僅在具台股市場資料模組時產出 | 僅用相符美股市場資料 | 不得把 A 股複盤套入台股。 |

Fugle 僅供台灣上市／上櫃與 ETF 之行情資料，不用於美股；它不取代 MOPS、交易所公告、法人籌碼或基本面查核。

## 3. 資料品質門檻

每份報告須揭露：資料來源、資料日期、盤中或盤後、復權／非復權口徑，以及下列品質之一：

- **已驗證**：市場、代碼、日期與主要價格已交叉核對。
- **部分資料**：缺法人、基本面或事件資料；只能列出缺口與觀察結論。
- **備援／延遲**：已改用備援來源或資料逾時；不可提供精確的進場、目標或部位結論。
- **不可用**：停止判定，明列待補資料。

MA 使用收盤價簡單移動平均；RSI 標示週期（預設 RSI(14)）；KD 標示 9,3,3；MACD 標示 12,26,9。自行計算時必須明標「自行計算值」。

## 4. 個股／ETF分析契約

每一份正式分析至少包括：

1. **核心結論**：趨勢、Stage 階段、信號型態、短線或 3–6 個月波段適配度、信心與最大風險。
2. **數據觀點**：價格與日期、MA5／10／20／60、量價、MACD、KD、RSI、支撐／壓力／Pivot、波動與多週期方向。
3. **情報分析**：可追溯的公告、營收、財報、法說與事件；ETF 不得套用個股財報推論。
4. **條件式操作計畫**：第一／第二進場條件、停損、加減碼、退出與失效條件、建議部位區間。

加碼只可在原始投資邏輯未破壞、支撐有效、轉強訊號與風險界線明確時提出；不得因低於成本而無條件攤平。

## 5. SEPA、Stage 2、VCP 與風險

- Stage 2 必須同時檢查中期均線上揚、均線結構、突破量能、相對強弱與過度延伸；僅上漲不足以判定。
- VCP 必須確認修正幅度與成交量收斂、波動下降、Pivot 清楚且突破放量；證據不足時寫「疑似 VCP，尚未確認」。
- 支撐／壓力必須說明依據：前高低、平台、缺口、均線、成交密集區或大量 K 棒。
- 目標價只可作保守／基準／樂觀情境推估，附推導方法與失效條件；不可當作事實。

## 6. 盤中、盤後與圖表

盤中資料必須標示時間，且不得稱為正式收盤；盤後才可使用完整日 K 與成交量。圖表或「AI 戰情儀表板」僅在使用者明確要求產圖時建立，採 16:9、繁體中文，且所有數字均須有來源、資料不足或自行計算標記。沒有真實回測樣本時，禁止虛構機率、勝率或相似走勢統計。

## 7. Fugle 在 JEAC 的角色

- FUGLE_API_KEY 存於 GitHub Repository secret，執行時才注入；禁止寫進 .env 範例、程式碼、報告或 issue／PR。
- FUGLE_PRIORITY=0（Repository variable，可省略）使 Fugle 成為台股第一個日線來源；失敗會降級到現有 YfinanceFetcher。
- 基本方案可用日線與盤中資料；Snapshot 與官方技術指標端點需使用 Fugle 提供該權限的方案。JEAC 的 MA／RSI／KD／MACD 仍由取得的 OHLCV 以統一口徑自行計算。
- API 失敗、額度不足、非交易日或代碼市場不符時，日誌必須明示來源與降級結果，不可靜默把資料當成台股官方收盤資料。
